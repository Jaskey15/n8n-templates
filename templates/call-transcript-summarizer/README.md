# Call Transcript Summarizer

Automatically summarizes inbound and outbound phone calls, then posts the summary to the contact's notes in GoHighLevel. Uses the GHL call transcript API and GPT to generate structured summaries with follow-up SMS drafts.

## How It Works

1. GHL fires a webhook when a call completes
2. Workflow finds the conversation and extracts the call message
3. Fetches the call transcript from GHL's transcription API
4. Formats the transcript with speaker labels (employee name lookup)
5. GPT generates a structured summary with performance notes and a follow-up SMS draft
6. Summary is posted as a note on the contact record

## Prerequisites

- [n8n](https://n8n.io) instance (self-hosted or cloud)
- [GoHighLevel](https://www.gohighlevel.com/) account with call tracking and transcription enabled
- [OpenAI](https://platform.openai.com/) API key

## Setup

### 1. Import Workflow

Import `workflows/call-transcript-summarizer.json` into n8n.

### 2. Replace Placeholders

| Placeholder | Where to Find Your Value |
|-------------|-------------------------|
| `YOUR_GHL_CREDENTIAL_ID` | n8n credentials page |
| `YOUR_OPENAI_CREDENTIAL_ID` | n8n credentials page |
| `YOUR_WEBHOOK_ID` | Auto-generated by n8n on import |
| `YOUR_WEBHOOK_PATH` | Choose your own webhook path |
| `TEMPLATE_WORKFLOW_ID` | Auto-assigned by n8n on import |

### 3. Configure Employee Lookup Map

In the "Format Transcript" code node, update the `EMPLOYEE_NAMES` map with your team's GHL user IDs and names:

```javascript
const EMPLOYEE_NAMES = {
  'YOUR_EMPLOYEE_1_USER_ID': 'Employee 1',
  'YOUR_EMPLOYEE_2_USER_ID': 'Employee 2',
  'YOUR_EMPLOYEE_3_USER_ID': 'Employee 3',
  'YOUR_EMPLOYEE_4_USER_ID': 'Employee 4'
  // Add more employees here
};
```

Find employee user IDs via the GHL API (`/users/` endpoint) or by inspecting call message data.

### 4. Customize the Summarization Prompt

The system prompt in the "Summarize Call" node provides a high-level summary structure (Service & Context, Goals, Pricing, Next Steps, Performance Notes, Follow-Up SMS). Customize it to match your sales process:

- Adjust the summary sections to fit the information your team cares about
- Modify the follow-up SMS tone and style to match your brand
- Add any formatting constraints specific to your CRM (e.g., character limits, encoding restrictions)

### 5. Set Up GHL Webhook

Create a GHL workflow that fires a webhook to your n8n instance when a call completes. The webhook payload should include:

- `contact_id` - the GHL contact ID
- `customData.first_name` - the customer's first name (optional)

### 6. Activate

Activate the workflow in n8n.

## Summary Output Format

The generated summary includes:

- **Service & Context** - Vehicle/asset details, relevant context
- **Goals & Priorities** - Customer's desired outcome, specific concerns
- **Offer & Pricing** - Package discussed, comparison shopping
- **Next Steps** - Follow-up plan with dates
- **Performance Notes** - Strengths and improvement areas for the rep
- **Follow-Up SMS Draft** - Ready-to-send text with assumptive close
